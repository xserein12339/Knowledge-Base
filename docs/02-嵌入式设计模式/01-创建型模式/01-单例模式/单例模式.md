# 单例模式

​	在嵌入式系统开发中，单例模式（Singleton Pattern）是一种非常实用的设计模式。它的核心思想是**确保一个类在系统中只有一个实例，并提供一个全局的访问点**

## 1. 单例模式使用场景

### 1.1 硬件资源管理

​	每个硬件外设只能创建一个实例，不能同一个外设由多个实例控制。单例模式能确保这些硬件资源被统一管理，避免重复初始化和配置冲突

#### 1.1.1 串口外设单例实现示例

(1)**饿汉实现**

```c
/**
 * @brief UART句柄
 */
typedef struct uart_t {
    UART_TypeDef *regs;
    uart_config_t cfg;
    bool is_inited;
} uart_t;

/* 饿汉创建实例 */
static uart_t g_uart_instances[UART_INSTANCE_MAX] = {
    [UART_INSTANCE_1] = {.regs = UART1, .is_inited = false},
    [UART_INSTANCE_2] = {.regs = UART2, .is_inited = false},
    [UART_INSTANCE_3] = {.regs = UART3, .is_inited = false},
};

uint8_t uart_get_instance(uart_instance_t id, uart_instance_handle_t *handle)
{
    if (handle == NULL) {
        return 1U;
    }

    if ((uint32_t)id >= (uint32_t)UART_INSTANCE_MAX) {
        *handle = NULL;
        return 1U;
    }
    *handle = &g_uart_instances[id];
    LOG_I("get instance %d, regs=%p", (int)id, (void *)g_uart_instances[id].regs);
    return 0;
}
```

优点：

* **无竞态条件**：无需互斥量保护初始化过程。
* **启动时序确定**：内存布局由链接脚本决定，不依赖调用顺序。
* **符合 MISRA-C / AUTOSAR**：避免运行时动态初始化。

缺点：

* 占用固定 RAM，即使实例从未被使用。
* 

(2)**懒汉实现**

```c
/**
 * @brief UART句柄
 */
typedef struct uart_t {
    UART_TypeDef *regs;
    uart_config_t cfg;
    bool is_created;
    bool is_inited;
} uart_t;

/* 实例池 */
static uart_t g_instances[UART_INSTANCE_MAX];

/**
 * @brief 根据实例ID获取UART寄存器基地址
 * 
 * @param id UART实例编号 
 * 
 */
UART_TypeDef* uart_get_regs(uint8_t id)
{
    if (id >= UART_INSTANCE_MAX) {
        return NULL;
    }
    /* 返回对应mock regs */
    static UART_TypeDef g_mock_uarts[UART_INSTANCE_MAX] = {
        [UART_INSTANCE_1] = { .SR = UART_SR_TXE_Msk, .DR = 0 },
        [UART_INSTANCE_2] = { .SR = UART_SR_TXE_Msk, .DR = 0 },
        [UART_INSTANCE_3] = { .SR = UART_SR_TXE_Msk, .DR = 0 },
    };
    return &g_mock_uarts[id];
}

uint8_t uart_get_instance(uart_instance_t id, uart_instance_handle_t *handle)
{
    if (handle == NULL) {
        return 1U;
    }
    if ((uint32_t)id >= (uint32_t)UART_INSTANCE_MAX) {
        *handle = NULL;
        return 1U;
    }
    if (!g_instances[id].is_created) {
        g_instances[id].regs = uart_get_regs(id);   ///< 调用时创建实例
        if (NULL == g_instances[id].regs) {
            *handle = NULL;
            return 2U;
        }
        g_instances[id].is_inited = false;
    }
    g_instances[id].is_created = true;
    *handle = &g_instances[id];
    LOG_I("get instance %d, regs=%p", (int)id, (void *)(*handle)->regs);
    return 0;
}

```

优点：

* 按需分配，节省未使用实例的 RAM。

缺点：

* **多线程复杂度高**：必须引入 RTOS 互斥量或原子操作，裸机环境需用临界区，
* **启动不确定性**：首次访问的时间点不可预测，影响实时性分析。
* **安全认证障碍**：DO-178C / ISO 26262 等标准通常禁止运行时动态初始化关键组件。
