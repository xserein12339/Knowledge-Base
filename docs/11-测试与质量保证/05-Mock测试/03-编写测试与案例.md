# 03 — 编写测试与案例

## 1. 测试四步法（Arrange → Act → Assert）

每个测试用例遵循三段式结构：

```c
void test_example(void)
{
    // 1. Arrange — 配置 Mock 行为 + 初始状态
    gpio_config_fake.return_val = 0;   // 模拟底层返回 ESP_OK

    // 2. Act — 调用被测函数
    int ret = pal_gpio_set_direction(4, PAL_GPIO_DIR_INPUT);

    // 3. Assert — 验证结果
    TEST_ASSERT_EQUAL(0, ret);
    TEST_ASSERT_EQUAL_UINT(1, gpio_config_fake.call_count);
}
```

## 2. 常用断言（Unity API）

### 基础断言

```c
TEST_ASSERT_EQUAL(expected, actual);       // 相等（自动类型推导）
TEST_ASSERT_EQUAL_INT(expected, actual);   // int 比较
TEST_ASSERT_EQUAL_UINT(expected, actual);  // unsigned int 比较（call_count 必须用此）
TEST_ASSERT_EQUAL_HEX(expected, actual);   // 十六进制比较
TEST_ASSERT_TRUE(condition);               // 布尔真
TEST_ASSERT_FALSE(condition);              // 布尔假
TEST_ASSERT_NULL(ptr);                     // 指针为空
TEST_ASSERT_NOT_NULL(ptr);                 // 指针非空
```

**关键**：`gpio_config_fake.call_count` 是 `unsigned int`，必须使用 `TEST_ASSERT_EQUAL_UINT`，否则 Unity 的类型推导可能出错。

### 字符串 / 内存

```c
TEST_ASSERT_EQUAL_STRING(expected, actual);
TEST_ASSERT_EQUAL_MEMORY(expected, actual, len);
```

### 预期失败（测试"不该到达的路径"）

```c
TEST_FAIL();          // 无条件失败
TEST_FAIL_MESSAGE("不应该执行到这里");
```

## 3. FFF Mock 控制 API

### 基础控制

```c
// 重置单个 Fake 的所有状态（通常在 setUp 中调用）
RESET_FAKE(gpio_config);
// 展开为: memset(&gpio_config_fake, 0, sizeof(gpio_config_fake));

// 设置固定返回值
gpio_config_fake.return_val = 0;            // 永远返回 ESP_OK
gpio_config_fake.return_val = ESP_ERR_NO_MEM; // 永远返回内存不足

// 设置序列返回值（每次调用返回下一个值）
gpio_config_fake.return_val_seq_len = 3;
gpio_config_fake.return_val_seq[0] = 0;     // 第 1 次调用 → ESP_OK
gpio_config_fake.return_val_seq[1] = 0;     // 第 2 次调用 → ESP_OK
gpio_config_fake.return_val_seq[2] = -1;    // 第 3 次调用 → ESP_FAIL
```

### 调用验证

```c
// 调用次数
gpio_config_fake.call_count          // 总调用次数

// 参数值（最近一次调用的参数快照）
gpio_config_fake.arg0_val            // 第 1 个参数的值
gpio_config_fake.arg1_val            // 第 2 个参数的值（仅 FAKE_VALUE_FUNC2+）

// 参数历史
gpio_config_fake.arg0_history[0]     // 第 1 次调用的第 1 个参数
gpio_config_fake.arg0_history[1]     // 第 2 次调用的第 1 个参数
```

### 自定义函数行为

```c
// 当 return_val 不够灵活时，可写 custom_fake
int my_gpio_config_custom(const gpio_config_t *cfg) {
    // 例如：根据 pin_bit_mask 返回不同值
    if (cfg->pin_bit_mask & (1 << 99))  // 非法引脚
        return ESP_ERR_INVALID_ARG;
    return ESP_OK;
}
gpio_config_fake.custom_fake = my_gpio_config_custom;
```

## 4. 实战：pal_gpio 全量测试

以下从 pal_gpio 的 22 个测试中提取代表性用例。

### 4.1 正向路径 — 配置引脚为输出

```c
/**
 * @brief 设置引脚为输出模式，应调用 gpio_config 且 mode 为 OUTPUT
 */
void test_set_direction_output(void)
{
    gpio_config_fake.return_val = 0;

    int ret = pal_gpio_set_direction(12, PAL_GPIO_DIR_OUTPUT);

    /* 验证返回值透传 + 调用次数 */
    TEST_ASSERT_EQUAL(0, ret);
    TEST_ASSERT_EQUAL_UINT(1, gpio_config_fake.call_count);
}
```

**验证点**：返回值正确传递；底层函数确实被调用了。

> ⚠️ 这里**不能**写 `TEST_ASSERT_EQUAL(GPIO_MODE_OUTPUT, gpio_config_fake.arg0_val->mode)`，
> 因为 `arg0_val` 指向 `pal_gpio_set_direction` 栈上的局部变量 `cfg`，
> 函数返回后该内存区域可能被覆盖。

### 4.2 错误路径 — 底层返回失败

```c
/**
 * @brief gpio_config 底层返回错误时，PAL 应透传错误码
 */
void test_set_direction_fail(void)
{
    gpio_config_fake.return_val = ESP_ERR_INVALID_ARG;

    int ret = pal_gpio_set_direction(8, PAL_GPIO_DIR_INPUT);

    TEST_ASSERT_EQUAL(ESP_ERR_INVALID_ARG, ret);
}
```

**验证点**：PAL 不吞掉错误码，原样透传给上层。

### 4.3 翻转逻辑

```c
/**
 * @brief 从低电平翻转为高电平
 */
void test_toggle_from_low(void)
{
    gpio_get_level_fake.return_val = 0;   // 当前低
    gpio_set_level_fake.return_val = 0;

    int ret = pal_gpio_toggle(10);

    TEST_ASSERT_EQUAL(0, ret);
    /* 验证 set_level 被调用，且参数为 1（高） */
    TEST_ASSERT_EQUAL(1, gpio_set_level_fake.arg1_val);
}
```

### 4.4 幂等操作 — ISR 服务重复安装

```c
/**
 * @brief ISR 服务安装应是幂等的，第二次不重复调用底层
 */
void test_install_isr_service_twice_should_be_idempotent(void)
{
    gpio_install_isr_service_fake.return_val = 0;

    /* 第一次安装 */
    int ret = pal_gpio_install_isr_service(0);
    TEST_ASSERT_EQUAL(0, ret);
    TEST_ASSERT_EQUAL_UINT(1, gpio_install_isr_service_fake.call_count);

    /* 第二次应跳过，不再调用底层 */
    ret = pal_gpio_install_isr_service(0);
    TEST_ASSERT_EQUAL(0, ret);
    TEST_ASSERT_EQUAL_UINT(1, gpio_install_isr_service_fake.call_count);
}
```

**验证点**：PAL 内部的 `static bool g_isr_service_installed` 正确拦截了重复调用。

### 4.5 中断配置 — 禁用不应注册 ISR

```c
/**
 * @brief 设置中断为 DISABLE 时不应注册 ISR handler
 */
void test_set_intr_disable_should_not_add_handler(void)
{
    gpio_set_intr_type_fake.return_val = 0;

    int ret = pal_gpio_set_intr(3, PAL_GPIO_INTR_DISABLE,
                                dummy_isr_cb, NULL);
    TEST_ASSERT_EQUAL(0, ret);
    /* 中断类型被设置为 DISABLE，但 ISR handler 未注册 */
    TEST_ASSERT_EQUAL_UINT(1, gpio_set_intr_type_fake.call_count);
    TEST_ASSERT_EQUAL_UINT(0, gpio_isr_handler_add_fake.call_count);
}
```

### 4.6 setUp / tearDown 模板

```c
void setUp(void)
{
    /* 每个测试前重置所有 Fake 到零状态 */
    RESET_FAKE(gpio_config);
    RESET_FAKE(gpio_set_pull_mode);
    RESET_FAKE(gpio_set_level);
    RESET_FAKE(gpio_get_level);
    RESET_FAKE(gpio_set_drive_capability);
    RESET_FAKE(gpio_install_isr_service);
    RESET_FAKE(gpio_isr_handler_add);
    RESET_FAKE(gpio_isr_handler_remove);
    RESET_FAKE(gpio_intr_enable);
    RESET_FAKE(gpio_intr_disable);
    RESET_FAKE(gpio_set_intr_type);
}

void tearDown(void)
{
    /* 重置 PAL 层内部 static 状态，确保测试用例完全隔离 */
    pal_gpio_uninstall_isr_service();
}

int main(void)
{
    UNITY_BEGIN();
    RUN_TEST(test_set_direction_input);
    RUN_TEST(test_set_direction_output);
    RUN_TEST(test_set_direction_fail);
    /* ... 更多 RUN_TEST ... */
    return UNITY_END();
}
```
